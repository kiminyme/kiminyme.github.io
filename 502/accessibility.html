Web Accessibility for Users with Low Vision

Click this text to jump to a list of Websites with more information.

This page is designed for a beginning course in Web Design using HTML and CSS at the university level.

The following information explains what considerations should be taken into account when designing a Web page to ensure that the information is readily available to a user with limited or no vision. Many of these steps will be useful to all visitors of the Web site.

There are three categories of users that these steps will help:

  • Persons with reading disabilities who may use a screen reader or other assistive devices.
  • Persons with low vision who can see, but who cannot see well enough to read content easily without assistive technology.
  • Persons with no vision at all, who depend on a screen reader or Braille keyboard, and who cannot use a mouse or other pointing device.

For visitors with low vision

Remember that not all people with limited vision are completely blind. Improve your Web design by making the content more visible overall.

Use maximal contrast. A visitor with limited vision may have problems distinguishing color sets readily. To this end, the designer of the Web page should focus on making sure that there is sufficient contrast between the text (foreground) colors and the background colors. Use dark colors on a light background or vice versa.

Use standard serif or sans serif fonts. Script or handwriting fonts may be difficult to read.

Use standard font sizes. If the designer uses font definitions that force the letters to be too small, it may be hard for a visitor to change the settings in their browser to display a larger font. Using percentage or relative sizes rather than set pixel sizes makes the settings more flexible for everyone.

Use a standard navigation bar with the same links in the same order on each page. This will help all visitors more easily navigate from one page to the next.

Avoid including meaningful text as graphics files. Screen readers cannot read text that is embedded in a graphic element, and this text will not size easily for users who use magnification.

Consider including an alternate CSS page with styles that present the content in a more linear fashion. A no-style option is built into the View menu in current versions of Firefox and Internet Explorer, and it may be useful to preview your page using that option. An alternate style page will give you more control of how the information flows on the page than the no-style option does.

For visitors who use screen readers

Keep in mind that not all visitors who use screen readers are blind. People with reading problems like dyslexia will be able to see the page, so it should still be visually interesting. However, you should also design for users who cannot use a mouse or other pointing device, either because they cannot see where the device is pointing, or because they cannot hold a pointing device.

Pay attention to the source code.

  • DO Include text descriptions of any pictures or visual elements with meaningful content that are included in the page. If the picture is a hyperlink, include a description of what the link will open.
  • DON'T include content that requires a mouse or other pointing device to see the content, like menus that open only when the user points to them.
  • DO provide links that allow users to skip over long scripts or other text elements in the source code that do not provide meaningful content for the page.
  • DON'T use colors alone to convey meaning. This will be meaningless to someone who can't see colors at all.
  • DO include a link at the top of the page to allow the user to skip the navigation bar at the top and jump straight to the main content of the page.
  • DON'T include too many hyperlinks on the same page. Grouping content and using subgroups can help limit the amount of content that a blind user has to scan to find the information they are looking for (and it can help even visitors with normal vision navigate faster to a specific topic within the Web site.

For more information:

AAO.org: Ensure your website is accessible to patients with low vision:
This page is written for the American Association of Opthalmology, for general information about how low vision can affect accessibility to the information in a Web page.

A List Apart: Big, Stark & Chunky:
This page is written for a Website devoted to sharing information about developing and using the Web. The author advocates creating a separate CSS page that is geared toward visitors with low vision, so that users can switch to a more vision-friendly version of the page.

Guidelines for Accessible and Usable Web Sites: Observing Users Who Work With Screen Readers:
This site gives recommendations for Web page design based on observations and surveys of how users use screen readers to navigate Web pages.

Low Vision Assistive Technologies in Web Design:
This site presents techniques that can be used to make a Web page more user-friendly to Braille monitors and other visitors with low vision. It also provides links to online tools that can be used to verify that the site is designed appropriately for visitors with vision problems.

WebAim: Visual Disabilities:
This site discusses what low vision is, and its various causes. It gives recommendations for Web design that can be useful for anyone with any form of low vision or blindness.
